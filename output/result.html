Introduction

This document is intended for all people who would like to obtain an XSLT Trainee/Developer position at Intelliarts company. The document contains a test problem for applicants, a short introduction into DocBook and many links to resources where you can learn more about technologies you are supposed to work with: XML, XSLT, HTML, XPath, XQuery and so on.

The test problem contains a few levels. We assume that one week should be enough to learn all required technologies and develop a solution for the test problem. In case you complete this work faster then you can implement solutions for optional and bonus tasks. The more levels you complete, the better your chances are.

Problem description

The final goal of the test problem is to create an XSLT transformation which converts an XML document into HTML document. With the test problem description you get the source of this document in the XML format, or strictly speaking in DocBook, which is semantic markup language for technical documentation.

Chapters below contain a brief description of several DocBook elements (or group of elements) and a few examples, thus you can easily use this document to test your transformation.

DocBook elements catalog

This chapter contains descriptions of the most commonly used DocBook elements with samples. We don't consider structural elements like book, article or chapter, since for the test problem we chose only the article element.

Block-level elements

Block-level elements in DocBook are elements like paragraphs, lists, and so forth. In the result HTML document they are usually represented with block-level elements. Next chapters briefly describe some of the block-level elements and provide ideas about how they can be rendered.

Paragraph element

Element name: para.

Description: the para element represents a paragraph in the document. Usually this element is rendered as a block with padding before and after it.

Example:

Lorem ipsum dolor sit amet, consectetur adipisicing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua. Ut enim ad minim veniam, quis nostrud exercitation ullamco laboris nisi ut aliquip ex ea commodo consequat. Duis aute irure dolor in reprehenderit in voluptate velit esse cillum dolore eu fugiat nulla pariatur. Excepteur sint occaecat cupidatat non proident, sunt in culpa qui officia deserunt mollit anim id est laborum.

Ordered list element

Element name: orderedlist.

Description: ordered list is a list of items where each list item is marked with a number. Ordered lists can be used for describing a procedure which contains a number of steps. Ordered list is marked with orderedlist element. Every list item in the list is represented by the listitem element. Content of each list item is wrapped by a para element. Besides, the orderedlist element may also contain an optional para element in the beginning to describe the content of the list.

Examples.

The following list is a simple ordered list with four items. By default Arabic numbers are used to mark list items.

  1. Lorem ipsum dolor sit amet, consectetur adipisicing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua.

  2. Ut enim ad minim veniam, quis nostrud exercitation ullamco laboris nisi ut aliquip ex ea commodo consequat.

  3. Duis aute irure dolor in reprehenderit in voluptate velit esse cillum dolore eu fugiat nulla pariatur.

  4. Excepteur sint occaecat cupidatat non proident, sunt in culpa qui officia deserunt mollit anim id est laborum.

The next list has a specific list intro element and 3 steps. List intro is a block level element (similar to paragraph) and should be appropriately indented (user should see that description corresponds to the list).

    Lorem ipsum dolor sit amet, consectetur adipisicing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua.

  1. Ut enim ad minim veniam, quis nostrud exercitation ullamco laboris nisi ut aliquip ex ea commodo consequat.

  2. Duis aute irure dolor in reprehenderit in voluptate velit esse cillum dolore eu fugiat nulla pariatur.

  3. Excepteur sint occaecat cupidatat non proident, sunt in culpa qui officia deserunt mollit anim id est laborum.

List elements can contain different block/inline level elements. The following list has several different elements inside every item. Such example of content shouldn't be a problem if your transformation is flexible and expandable.

  1. Lorem ipsum dolor sit amet, consectetur adipisicing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua.

  2. Ut enim ad minim veniam, quis nostrud exercitation ullamco laboris nisi ut aliquip ex ea commodo consequat.

    • Ut enim ad minim veniam, quis nostrud exercitation ullamco laboris nisi ut aliquip ex ea commodo consequat.

Unordered list element

Element name: itemizedlist.

Description: the itemizedlist element is used to add a simple unordered list to the document. Each list item is represented by a single listitem element. All list item content is wrapped by a single para element. Also a list may contain an optional para element at the beginning to describe contents of the list.

Formatting: the list should be rendered as a simple bulleted list. When a list intro element is available it should be properly indented to let user see that it is a part of the list.

Examples:

Simple unordered list that contains four items:

  • Lorem ipsum dolor sit amet, consectetur adipisicing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua.

  • Ut enim ad minim veniam, quis nostrud exercitation ullamco laboris nisi ut aliquip ex ea commodo consequat.

  • Duis aute irure dolor in reprehenderit in voluptate velit esse cillum dolore eu fugiat nulla pariatur.

  • Excepteur sint occaecat cupidatat non proident, sunt in culpa qui officia deserunt mollit anim id est laborum.

Unordered list with a list intro and three list items:

    Lorem ipsum dolor sit amet, consectetur adipisicing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua.

Inline elements

Inline-level tags are elements like emphasis, hyperlinks, and so forth. They wrap a text within a block-level element. Usually the content of these elements is highlighted with a different font (different font family, font style, font weight etc).

'command' element

Element name: command.

Description: the keycap element represents a text printed on a key on a keyboard.

Formatting: the content of this element should be formatted inline, the key caption is usually rendered with bold font.

Example:

Press F1 to see a help window.

'emphasis' element

Element name: emphasis.

Description: the emphasis element provides a method for indicating that certain text should be stressed in some way.

Formatting: the content of this element should be formatted inline, emphasized text is usually rendered with italic font.

Example:

Here is the thing we would like to emphasize.

Optional inline elements task

Create a document with a fragment where emphasis and keycap elements are combined. Verify if your transformation supports them, extend it if needed.

Link elements

Link tags are used to create links which lead to another part of the document or to any external resource, for example a website.

Internal links

Element name: link.

Description: the link element is used to organize links to other parts of the document. Links are organized using the id attribute. Each DocBook element can have an id attribute. To create a link to the particular element the linkend attribute value of the link element should contain the same value as the id attribute of the target element. The text within the link element is the link text.

Formatting: the links text should be rendered with underlined font in green color.

Example:

Please click here to go to the Introduction chapter.

Links without a correct target set, like this, should be highlighted but shouldn't take the reader to another place.

External links

Element name: ulink.

Description: the ulink element is used to organize links to external resources, e.g. websites. The url attribute of the ulink element should contain an URL of a site.

Formatting: the link text should be rendered with underlined font in green color.

Example:

Please click here to read more about external links in DocBook.

Optional link elements task

In addition to the general links handling, implement endterm attribute support. The endterm attribute specifies the location of the text that should be used as a link label.

Example:

The following link - link example - should have a label with text "Inline elements".

It would also be nice to add support of rendering other inline elements inside links (e.g. keycap, emphasis).

Example:

The following link - bold or italic - should be rendered using corresponding fonts.

Bonus tasks

If you manage to complete the basic level transformation faster than a week, here you can find a few bonus tasks. The complexity of bonus tasks is increased, so it makes sense to work on them in the provided order.

Implement table of contents

Extend the transformation to make it insert the table of contents (TOC) in the beginning of the document.

  1. TOC should have a list of all sections/titles encountered in the document

  2. TOC entries should have appropriate numbering and indentation regarding their position in the document hierarchy.

  3. TOC items should be links to the appropriate section/title, so when you click a title in the TOC, you are redirected to the corresponding section.

Implement multiple HTML page output

Extend the transformation to make it generate multiple HTML pages for the input document.

  1. Each document section should be presented as a separate HTML page.

  2. There should be an index.html page that contains the TOC.

  3. A click on a TOC entry should open the corresponding section in the same window

  4. If a section has child sections, then the corresponding page should contain mini-TOC names "Child Topics". Mini-TOC should have appropriate titles/links of/to child sections.

  5. Each page should have a navigation block at the top with links to previous, parent and next sections.

Tools

It doesn't matter which XSLT processor you use. However, we want to let you know that we are working in the Java environment and use Saxon XSLT processor in most of our projects.

It would be nice if you provide a simple Ant build which allows you to run a transformation.

Acceptance criteria

The acceptance procedure is very simple. We run you transformation to transform the source of this document, then review the resulting HTML document.

The transformation should be completed without errors. The resulting document should contain all the contents of the original document formatted accordingly.

Resources

This chapter contains links to various useful resources you may need to learn more about XML, XSLT, HTML, XPath, XQuery and others. You can use any information sources you like (not listed here), but we decided to share with you some our favourite ones.

All technologies (except DocBook) you are supposed to learn are developed by World Wide Web Consortium (W3C), so you can find a lot of information, including specifications and tutorials at the W3C related sites:

  • https://www.w3.org/TR/ - main W3C site where all specifications, a lot of tutorials and links to other resources are available;

  • http://www.w3schools.com/ - site with a lot of tutorials, examples and useful stuff.

XML

XML (eXtensible Markup Language) is a set of rules for encoding documents electronically. It is defined in the XML 1.0 Specification produced by the W3C and several other related specifications; all are fee-free open standards. You can find a lot of information about XML at:

  • http://en.wikipedia.org/wiki/XML - Wikipedia page about XML, contains basic information about XML and a lot of links to useful XML-related resources.

  • https://www.w3schools.com/xml/default.asp - w3schools XML tutorial. Here you can find useful information about XML and XML-related technologies like XPath, XQuery and get info about XML DOM.

XSLT

XSLT (XSL Transformations) is a declarative, XML-based language used for the transformation of XML documents into other XML (or non-XML) documents.

  • http://en.wikipedia.org/wiki/XSLT - Wikipedia page about XSLT, provides basic information about XSLT and a lot links to useful XSLT-related resources.

  • http://www.zvon.org/xxl/XSLTutorial/Output_rus/index.htm - one of the best XSLT tutorials/references with a lot of samples (in many languages including Russian).

  • http://xmlhack.ru/books/xslt - probably the best book about XSLT 1.0 in Russian (partially available).

HTML

HTML, which stands for Hyper Text Markup Language, is the predominant markup language for web pages. It provides the means to create structured documents by denoting structural semantics for text such as headings, paragraphs, lists etc as well as for links, quotes, and other items.

  • http://en.wikipedia.org/wiki/HTML - Wikipedia page about HTML, provides basic information about HTML and a lot links to useful HTML-related resources.

  • http://www.w3schools.com/html/default.asp - excellent tutorials, online editor which allows you to quickly check how things work.

DocBook

DocBook is a semantic markup language for technical documentation. It was originally intended for writing technical documents related to computer hardware and software but it can be used for any other sort of documentation.

  • http://www.docbook.org/ - the official DocBook home page.

  • http://en.wikipedia.org/wiki/DocBook - Wikipedia page about DocBook, provides basic information about DocBook and a lot links to useful DocBook-related resources.

  • https://tdg.docbook.org/tdg/4.5/docbook.html - DocBook: The Definitive Guide - very detailed book about DocBook format. Best place to search information about any element, DocBook document authoring and so on.

ANT

Apache ANT is a Java based build tool from Apache Software Foundation.

  • https://ant.apache.org/ - Apache ANT official home page. Here you can download ANT tool and find manual with required information about ANT tasks and how to run build from a command line.

  • https://www.tutorialspoint.com/ant/ - Apache ANT Tutorial for the beginners to help them understand basic functionality of Apache ANT tool.
